 EMAIL AS PRIMARY KEY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What this means:
✓ Email IS the username (no separate username field)
✓ Each email can only have ONE account
✓ Email uniquely identifies every user
✓ Email verification ensures validity
✓ Email cannot be duplicated in the system

Examples:
• john@example.com → ONE account with that email
• jane@example.com → ONE account with that email
• john@example.com again → ERROR (already exists)

 ERROR HANDLING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Wrong password               → Shows error: "Invalid login credentials"
