🗄️ DATABASE SCHEMA
═════════════════════════════════════════════════════════════════════════════════

TABLE 1: auth.users (Supabase Built-in)
───────────────────────────────────────
Columns:
  • id (UUID) - Primary key
  • email (TEXT) - UNIQUE, used for login
  • encrypted_password (TEXT) - Bcrypt hash
  • raw_user_meta_data (JSONB) - Stores name, contact_number, state, city
  • confirmed_at (TIMESTAMP) - Email confirmation status
  • created_at (TIMESTAMP) - Account creation date

Location in Portal: Authentication → Users


TABLE 2: public.users (Custom)
──────────────────────────────
Columns:
  • id (UUID) - Foreign key to auth.users(id)
  • email (TEXT) - UNIQUE, PRIMARY KEY for app
  • name (TEXT) - Full name from sign-up
  • contact_number (TEXT) - 10-digit phone
  • state (TEXT) - Indian state
  • city (TEXT) - City name
  • created_at (TIMESTAMP) - Account creation date

Location in Portal: SQL Editor → public → users
Security: Row Level Security (RLS) enabled


🔑 EMAIL AS PRIMARY KEY
═════════════════════════════════════════════════════════════════════════════════

Email is the UNIQUE IDENTIFIER:
  ✅ One email = One account
  ✅ Used for login (not separate username)
  ✅ Verified via confirmation email
  ✅ Stored as UNIQUE in both tables
  ✅ Matches UPSC registration norms

Example:
  Sign-up: john@example.com
  Login: john@example.com (email = username)
  Database: PRIMARY KEY in public.users
